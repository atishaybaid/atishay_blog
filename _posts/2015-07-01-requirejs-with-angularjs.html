Before diving into requirejs we will have to understand how script loaders work.

How script loaders work
When loading multiple scripts through <script> tag ,blocks the loading of the page until the JS code is available.Hence the script loaders such as require which lodes the script in  asynchronous manner has a advantage.



Why use  RequireJs
RequireJs is a javascript library which helps is lazy loading javascript dependencies.Traditionally if we don't use a library such as require for loading our code we would use <script> tags and would call all the required files in the desired order.That could be a tedious task when managing a large javascript application.As we will have to call all the files in the order to resolve the dependencies.,i.e which file to be loaded first and so on,also including all the files in our index page would hinder the performance as it would create too many open holes in our application.
To overcome this problem we could include all the code in a single file,but it would cause serious maintainability issues.
Hence we need a package dependency manger,which helps us loading the code

Types of Depencencies

  • Load :Which files or classes to be loaded and in what order
  • Construction:When classes are constructed,what instances are to be provided.
  • Run-time :What functions and objects do we need post instantiation.
  • AngularJS module dependency:One module depends on other module.(dependency injection)
RequireJS manages load and run time dependencies,angular manages construction and module dependencies issues,hence AngularJS injects instances and RequireJS injects classes.


RequireJS API'S
1)define():defines our dependencies and registers a callback factory function,which will be called when our dependencies are resolved.
2)require():callback function to be invoked when the defines() are completed.
3)config():Allows to configure source paths and aliases if any.
